## ❓ Interaktivní Kvízy

Testuj si své znalosti. Odpověď se zobrazí po kliknutí na tlačítko.

#### ❓ Otázka 1: Fabric Hierarchie

Kolik OneLake instancí existuje v jedné Microsoft Fabricu?

**A)** Jeden per Workspace

**B)** Jeden per Fabric (shared across all workspaces)

**C)** Jeden per Capacity

**D)** Libovolný počet (volitelně)

Zkontroluj

✅ Správně! OneLake je **jeden per Fabric**. Všechny workspaces sdílí jeden OneLake.

#### ❓ Otázka 2: Data Stores

Kterýkoliv Fabric data store (Lakehouse, Warehouse, Eventhouse) interně ukládá data v jakém formátu?

**A)** Libovolný formát (závisí na výběru uživatele)

**B)** Delta Parquet

**C)** Standard SQL format

**D)** Proprietary Microsoft format

Zkontroluj

✅ ✅ Správně! Všechna data v Fabricu = **Delta Parquet** pod kapotou (i když T-SQL a KQL jsou jen interfaces).

#### ❓ Otázka 3: Nástroje - Kdy Použít?

Máš CSV soubor, který chceš nahrát do Lakehouse a provést jednoduchou transformaci (filtrování, výběr sloupců). Jaký nástroj je NEJLEPŠÍ volba?

**A)** Data Pipeline + Copy Data Activity

**B)** Dataflow Gen2

**C)** Notebook + PySpark

**D)** Eventstream

Zkontroluj

✅ ✅ Správně! **Dataflow Gen2** je ideální pro low-code transformace. Pipeline + Copy by byla jen moves bez transformace.

#### ❓ Otázka 4: Real-Time Ingestion (RTI)

Máš data přicházející každou sekundu z IoT senzorů (teplota, vlhkost). Chceš agregaci: "Jaká byla průměrná teplota za poslední 5 minut?" Co použiješ?

**A)** Batch processing (denní batch)

**B)** Spark Structured Streaming + Tumbling Window (5 minut)

**C)** SQL Group By na Warehouse

**D)** Dataflow Gen2

Zkontroluj

✅ ✅ Správně! **Spark Structured Streaming + Tumbling Window** je optimální pro real-time agregace. Batch by byl pozdě.

#### ❓ Otázka 5: Jazyky

Který z těchto jazyků NENÍ v rozsahu DP-700?

**A)** T-SQL

**B)** PySpark

**C)** DAX

**D)** KQL

Zkontroluj

✅ ✅ Správně! **DAX** je pro Power BI (DP-600), ne pro DP-700. DP-700 se zaměřuje na data engineering, ne na BI.
